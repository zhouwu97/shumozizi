# 离线论文学习

此 Skill 只提升后续能力，不修改任一 `runs/<run-id>/state/run.json`，不阻断当前赛题，也不自动将历史方法套到新题。

1. 读取本地论文、原题和来源清点；若原题、页码或来源不明确，记录原因后停止。
2. 生成 `knowledge/cards/papers/<paper_id>.md`，记录题目类型、问题链、共享数学对象、模型选择理由、baseline、验证、图表叙事、摘要写法、可迁移模式、不可迁移内容、缺陷、复现风险和来源页码。
3. 不复制原论文的数字、结论、代码或未经许可的段落；不自动判断其“获奖”或“优秀”。
4. 更新可重建的索引，并保留来源、许可证与哈希信息。

生产解题时先做结构预检，再按论证读取最多三个能力包，并重新比较当前题的替代路线。

## 与生产论文的单向边界

本 Skill 不读取运行目录、不接收当前题的结果 ID，也不触发 `$mathmodel-paper`。它产出的论文卡始终是离线能力资产，而不是当前题的事实、数值或结论来源。

只有 production 结果已经冻结后，`$mathmodel-paper` 才可通过受限收据按需读取 1-2 张已登记卡，用于章节组织、模型解释的表达选择、验证叙事或 Figure Contract。收据只记录卡 ID、受控相对路径、索引/卡/来源哈希和允许用途；不复制卡正文，也不把本 Skill 与生产写作合并。

卡或其原论文中的数值、结论、代码、原始公式段和实验结果不得迁移到当前论文，且卡不能作为 citation、evidence、Claim-Evidence、结果引用或事实来源。当前论文的所有事实、图表、表格和结论只能来自本次 run 的 production/current/accepted 证据链。
