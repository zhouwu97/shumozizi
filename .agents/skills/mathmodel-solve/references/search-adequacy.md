# 非光滑优化的搜索充分性

先固定 exact objective；粗网格/中心点布尔时长只能是 approximate evaluator，不能称 proxy 或充当结论。搜索代理应输出距离约束的连续裕度，例如全表面最大视线距离的软化积分。

每个问题依次保留：基线恢复、全域低差异样本（含角度环绕和边界锚点）、独立校准、局部与全域候选、exact 重算和独立挑战。校准不能从同一 surrogate 的 top-k 抽取；至少包括独立分层/低差异样本、边界样本和事件邻域样本，并同时报告边际、投影和联合区域覆盖。校准至少报告：近似与代理的 false-zero、top-k recall/错序、相对基线的改善符号一致性，以及局部候选的代理变化。

稀疏目标中任意近似假零都先触发加密校准或重启，不能因为比例较小而通过。若代理将正 exact 候选判为零、top-k recall/改善一致性不足、局部代理平坦、真实协同变量的联合域覆盖不足，或挑战注入 incumbent，登记 `search_adequacy=failed`。挑战开始前应冻结 incumbent 的哈希/exact 值和可比阈值；候选冻结后独立 exact 重算 incumbent。挑战结果严格改善 incumbent，或在阈值不可事后下调的前提下达到该阈值，均可作为独立充分性证据；首次低于阈值只能做一次有界加密/换族复跑，仍失败才阻断。基线保持仅说明下界未丢失，不是质量通过；上游弱解不得收窄下游全域。
