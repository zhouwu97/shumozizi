# 建模可视化叙事

图表应回答一个会影响模型选择、求解可信度或结论边界的问题。它不是版面填充，也不重新决定已冻结的数值事实。

1. 读取能力路由、科学红队结论、当前结果和建模/实验报告。先写每张图的科学问题、真实数据或模型来源、结论影响和删除后会缺失的证据；同一张图可以承担多个问题。
2. 按证据角色选择表达，不规定固定图种：
   - 空间结构、轨迹、遮挡或场分布影响核心结论时，提供能看见对象、坐标、单位、轨迹和关键边界的三维模型证据；若二维或正交投影更可核验，应在分析中说明理由。
   - 优化问题提供“求解过程证据”和“搜索/最优性证据”的等价组合。启发式可用 best-so-far 与 proxy 校准/局部景观；MILP 可用界与 gap；枚举可用覆盖；解析解可用函数与边界证明。不要把日志序号伪装成收敛，也不要强制生成无意义的 3D 图。
   - 预测、网络、机理和评价题选择能检验拟合、残差、不确定性、流量、状态、敏感性或稳定性的图，而不是套用优化图。
3. 工具由证据任务决定：真实数据图可读取 `skills/3coding-visual/SKILL.md`；结构关系图可读取 `skills/4drawio/SKILL.md`；路由选择的 MATLAB/Octave 可用于独立图形核验和三维表达；科研模板仅在真实结果接口匹配时使用。模板演示数据绝不进入论文。
4. 将图脚本保存到 `code/figures/` 或 `code/matlab/`，输出写入 `figures/`。每张完成图都通过真实渲染收据登记，收据冻结的是命令实际执行、声明输入/脚本/输出的路径与哈希以及 PNG 可读性：

   ```powershell
   python scripts/figures/record_visualization.py runs/<run-id> `
     --input code/visualization-plan.json
   ```

   `production_result` 图只引用已放行的当前结果；模型示意和诊断图要如实标明范围，不能冒充数值结论。输入清单只是执行方声明，回执不能自动证明脚本读取了全部声明输入或没有读取未声明文件；`rendering_mode=3d` 也只是执行方声明，不能自动证明 PNG 真的是有科学价值的三维证据。
5. 图完成后由独立科学/视觉审查检查其实际内容是否呈现声明的数学对象和证据，而不只检查文件、标签或渲染模式。正文解释图如何支持模型、选择或限制；图注不能替代解释。

所有必要证据问题都由运行时按路由和等价模式验证后，才进入 `paper`。
