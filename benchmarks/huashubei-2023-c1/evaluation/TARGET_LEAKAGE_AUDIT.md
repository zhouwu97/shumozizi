# 目标泄漏核查

核查日期：2026-07-20

v3 的入睡方式模型将“入睡方式”纳入 one-hot 预处理器，同时又把它作为目标，构成直接目标
泄漏。旧结果 `q1-primary` 已通过不可变撤销记录标记为 `superseded`，替代结果为
`q1-primary-leakage-fixed-v4`。

| 响应 | 目标字段 | 输入字段 | 已排除字段 | 是否泄漏 |
| --- | --- | --- | --- | --- |
| 睡眠时长 | `sleep_hours` | 母亲年龄、妊娠周数、CBTS、EPDS、HADS、婴儿月龄、婚姻、教育、分娩方式、婴儿性别 | 原始/派生睡眠时长、睡醒次数、行为、入睡方式 | pass |
| 睡醒次数 | `睡醒次数` | 同上 | 原始/派生睡眠时长、睡醒次数、行为、入睡方式 | pass |
| 行为特征 | `婴儿行为特征` | 同上 | 原始/派生睡眠时长、睡醒次数、行为、入睡方式 | pass |
| 入睡方式 | `入睡方式` | 同上 | 原始/派生睡眠时长、睡醒次数、行为、入睡方式及其 one-hot 派生列 | pass |

所有四个 Pipeline 独立拟合预处理器。执行输出的 `feature_audit` 对每个响应登记目标、输入、
排除字段、目标派生字段和实际泄漏列；四项 `passed=true`。
