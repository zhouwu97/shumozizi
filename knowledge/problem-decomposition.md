# 问题拆解（按需参考）

来源：基于 `Lupynow/math-modeling-skills` 的问题分解思想独立整理；详见
[`THIRD_PARTY_NOTICES.md`](../../THIRD_PARTY_NOTICES.md)。本文件不替代当前题的阅读与判断。

1. 列出每一问的动词、输入、要求输出、硬约束和评分线索。
2. 画出问题依赖：共享数据、变量、参数、目标、约束和不确定性来源。
3. 为每问区分“需要预测、解释、优化、评价、反演还是决策”，不要仅按题号选算法。
4. 标明单位、数据时间边界、缺失机制、样本粒度和可能泄漏路径。
5. 只记录会改变模型或答案的歧义。若两种解释产生不同必答输出，才请求用户确认。

最小产物应当能说明：每问如何直接作答、哪些对象跨问复用、最早可以证伪哪一条路线。
