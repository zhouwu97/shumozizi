# 竞赛论文撰写

本 Skill 负责把当前运行的真实模型、结果和图表组织成论文。它不搜索网络、不查找同题答案、不复制优秀论文内容，也不生成没有证据的章节。

## 选择模板

在 `visualization -> paper` 前调用受控选择器，而不是手工复制模板或从 `default` 重建：

```powershell
python scripts/paper/select_template.py runs/<run-id> `
  --language <zh|en> --engine auto `
  --reason "比赛、语言与仓内模板匹配；优先 LaTeX。" --materialize
```

`auto` 优先 LaTeX。只有检测不到可用 LaTeX 编译器时才受控回退到 Typst，并将原因写入模板清单；用户明确选择 Typst 才传 `--engine typst`。显式选择不可用引擎或未知赛事会阻断，不能静默替代或手工搭建入口。

模板实例化会生成 `paper/sections/questions.tex` 或 `paper/sections/questions.typ`，其中的问题数量由本题 `required_questions` 动态决定。只编辑当前模板实际引用的入口和章节；不要假设固定三问或使用 `5_problem1.*` 一类旧文件名。旧正文和示例参考文献会被清空；标题、摘要、关键词等模板占位符会被保留为可检测项，必须以本题内容填写，终检拒绝其残留。

## 写作结构

先读取当前运行的分析报告、实验报告、真实图表和结果索引。每个问题应让读者能定位到：

1. 题意解释、对象和采用假设。
2. 变量、核心公式或推导，以及为什么该模型适用。
3. 实际求解或计算过程，而不是算法名称清单。
4. 当前运行的结果、验证、敏感性或适用边界。
5. 带对象、单位和范围的直接答案。

共享模型可以集中介绍，但每问的直接答案不能藏在总表、图注或前一问。公式、图和表只保留能帮助解释模型、选择、稳定性或限制的内容；不要按页数、图数或引用数凑材料。

图表必须来自 `figures/` 的真实输出。说明图回答的科学问题、数据或模型来源、它支持的结论和不能证明的内容。空间结构影响结论时，图中应能识别对象、坐标、单位和关键边界；搜索图不能把日志序号或同源代理当作最优性证明。

## 引用与编译

只引用实际采用的方法、定义或数据来源。LaTeX 使用 `\cite{key}` 与 `references.tex`；Typst 使用对应引用语法与 `references.typ`。离线论文卡仅可改善表达和组织，不能提供本题事实、数值、代码、段落或引用。

完成后编译：

```powershell
python scripts/paper/compile_paper.py runs/<run-id>
```

该命令冻结当前模板、源文件和 PDF 的编译回执。之后由独立新对话做匿名 PDF 盲审；修改正文、图表或 PDF 后必须重新编译和重新盲审。
