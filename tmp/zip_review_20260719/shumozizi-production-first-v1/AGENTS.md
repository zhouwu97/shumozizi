# AGENTS.md

## 总原则

你正在执行数学建模比赛生产任务，不是在建设审计系统。

优先级固定为：

```text
题意正确 > 模型合理 > 实验可信 > 论文完整 > 格式合规 > 内部工程记录
```

## 必读文件

开始前阅读：

- `docs/WORKFLOW_V1.md`
- `docs/MIGRATION_MAP.md`
- 当前题目目录中的全部官方材料

## 执行要求

1. 先生成 `work/problem_brief.md`，只在会改变数学问题或路线时请求人工确认。
2. 生成 `work/modeling_plan.md` 后进行建模审查。
3. 建模审查通过或带修复通过后立即编码，不等待所有文字材料完美。
4. 论文骨架在建模阶段建立，第一批可信结果出现后同步写正文。
5. 每个子问题独立形成“模型—代码—结果—图表—正文—检查”。
6. 关键数字只保留一个计算真源，论文通过自动值或明确结果键引用。
7. 普通比赛只进行一次独立终审；纯文字和排版修订使用差异复核。
8. 只有模型、数字、核心图表或结论改变时才重跑对应计算和独立终审。
9. 不在论文正文写 Gate、Ledger、Manifest、Reviewer、Prompt、Agent 等内部生产术语。
10. 不用增加 JSON、报告或状态字段来代替模型与论文内容。

## 禁止事项

- 默认启用 `formal_audit`
- 让作者逐问手工填写重复的 11 项准入矩阵
- 因为没有复杂模型就强行堆算法
- 因为运行预算有限就宣称全局最优或没有更优方案
- 把复算一致写成模型有效
- 等全部实验结束后才开始写论文
- 为了页数增加无论证价值的图、表和套话
- 用 AI 痕迹检测代替竞赛要求的 AI 使用披露

## 变更影响规则

| 变更类型 | 必须执行 |
|---|---|
| 错别字、措辞、段落顺序 | 编译、引用和视觉检查 |
| 图题、表题、解释文字 | 编译、图文一致性检查 |
| 数值或表格 | 对应子问题运行、检查、结果绑定、论文 |
| 假设、变量、模型或约束 | 对应子问题及依赖它的后续问题完整重跑 |
| 正式附件 | 附件生成、结果一致性和核包 |
| 模板与页边距 | 全文编译、页数、匿名和文件大小检查 |
