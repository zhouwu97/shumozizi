# 数学建模比赛生产主工作流 v1

## 1. 定位

本流程用于比赛当天和模拟赛的真实生产，不承担资格认证、长期能力封存或研究级审计。

目标是连续产出：

- 数学上可辩护的模型；
- 可复现的代码与结果；
- 完整、可独立阅读的论文；
- 符合当年规范的提交文件。

普通比赛只保留四个硬检查点：

```text
G0 题意与路线确认
G1 模型可执行且无根本矛盾
G2 关键结论有可复现证据
G3 最终论文与提交包合规
```

其余检查默认给出建议，不阻断持续生产。

## 2. 角色

### 作者任务

完成题意拆解、建模、编码、实验、图表和论文，并集中修订。

### 建模审查任务

只审题意、变量、假设、模型、目标、约束、基线和验证计划。

### 实验复现任务

只审输入、代码、实验、结果、图表和复现。

### 论文逻辑审查任务

只审逐问作答、模型论证、结果解释、图表价值、边界和可读性。

### 最终人工核包

只审当年格式、匿名、附件、AI 使用披露和提交文件一致性。

## 3. 阶段流程

## S0 题目解析与人工确认

生成 `work/problem_brief.md`：

- 子问题；
- 输入输出；
- 单位和范围；
- 强制约束；
- 正式附件；
- 关键歧义；
- 初步题型与路线。

只有会改变数学问题或路线的歧义才暂停。

### G0

人工确认没有理解错题、漏掉强制输出或采用错误口径。

## S1 建模方案与建模审查

生成 `work/modeling_plan.md`：

- 现实对象到数学对象的映射；
- 变量、参数和假设；
- 一个主模型；
- 一个廉价基线；
- 必要时的结构备选；
- 目标、约束和数学关系；
- 算法与停止条件；
- 验证计划；
- 图表计划；
- 子问题继承关系。

使用 `templates/review/model_review.md`。

结论：

```text
ACCEPT
ACCEPT_WITH_FIXES
REBUILD
```

只有根本性题意或模型矛盾允许 `REBUILD`。

### G1

模型可以进入实现，且不存在已知根本矛盾。

## S2 编码、实验与复现审查

每问目录：

```text
questions/<qid>/
├─ question.md
├─ model.md
├─ run.py
├─ check.py
├─ results/
│  ├─ result.json
│  ├─ tables/
│  └─ figures/
└─ paper.typ
```

`result.json` 是关键数值真源。

按题型选择最低证据：

- 预测：划分、泄漏、基线、误差、外推；
- 优化：可行性、约束、基线、搜索范围、停止原因、gap/bound；
- 评价：指标、归一化、权重、基线、稳定性；
- 仿真：量纲、初边值、稳定性、守恒、敏感性；
- 分类/聚类：不平衡、指标、稳定性、解释和失效样本。

使用 `templates/review/experiment_review.md`。

### G2

论文将使用的核心结论有可复现证据。非核心探索失败不阻断。

## S3 论文并行写作

论文不能等全部实验结束后才开始。

在 S1 后建立完整章节骨架；第一批可信结果出现后立即写问级正文。循环为：

```text
模型说明
→ 初次结果
→ 结果解释
→ 暴露缺口
→ 补实验或修模型
→ 更新正文
```

每问正文链：

```text
题目要求
→ 明确回答
→ 变量与假设
→ 数学表达
→ 求解方法
→ 核心结果
→ 验证/对照
→ 结果解释
→ 适用边界
```

## S4 论文逻辑审查

使用 `templates/review/paper_review.md`。

不再要求逐问手工填写 11 项准入矩阵。改为 `work/claim_evidence_map.md`：

```text
claim_id
正文位置
结果键/表/图
复现命令
适用边界
状态
```

审查顺序：

1. 逐问作答；
2. 模型能否独立阅读；
3. 结论是否由证据支持；
4. 基线、验证和边界；
5. 图表是否服务推理；
6. 摘要是否逐问给出方法、结果和结论；
7. 是否有夸大、虚构引用、无来源数字或内部语言；
8. 可读性。

## S5 独立终审与修订

完整稿只进行一次独立终审：

```text
READY_FOR_HUMAN_CHECK
MAJOR_REVISION
NOT_READY
```

修订后按影响范围复核：

| 变更 | 重跑范围 |
|---|---|
| 文字、错别字、格式 | 编译、引用、视觉 |
| 图题、表题、解释 | 编译、图文一致性 |
| 数字、表格 | 对应问计算、检查、结果绑定、论文 |
| 假设、模型、约束 | 对应问和依赖问完整重跑 |
| 正式附件 | 附件生成、核包、一致性 |

只有模型、数字或核心结论改变时才进行新的独立终审。

## S6 格式合规与提交核包

内容稳定后检查：

- 当年官方格式；
- 电子版第一页为摘要专用页；
- 摘要后强制分页；
- A4 和最小页边距；
- 正文页数；
- 不生成目录；
- 全文匿名；
- 正文引用与参考文献；
- 附录支撑材料文件列表；
- 全部完整可运行源程序；
- AI 使用正文标注、参考文献和 `AI工具使用详情.pdf`；
- 论文与支撑材料文件大小；
- PDF、支撑包和正式附件结果一致。

### G3

队员人工确认后提交。

## 4. 模式

| 模式 | 用途 | 规则 |
|---|---|---|
| `standard` | 正式比赛、模拟赛 | 四个硬检查点；一次独立终审 |
| `emergency` | 临近截止 | 保留题意、模型硬矛盾、核心证据和核包 |
| `formal_audit` | 赛后复盘、争议复现 | 额外启用冻结、Manifest、Validator、哈希和 Seal |

普通比赛禁止默认使用 `formal_audit`。

## 5. 时间分配

三天比赛建议：

- 题意与路线：10%–15%
- 建模、编码与实验：45%–55%
- 论文并行写作：25%–35%
- 终审、格式与核包：10%–15%

当内部报告数量超过论文核心章节数量，或审查耗时超过建模与写作耗时，应立即降级审查强度。
